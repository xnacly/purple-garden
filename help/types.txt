Purple garden provides the primitive types any programmer would expect:

    Void
    Bool
    Int
    Double
    Str

Furthermore, there are four non primitive types:

    Option<T>
    Array<T>
    Struct
    Foreign<identifier>


Binary operators
----------------

Both sides of a binary operator must have the same type. There is no
implicit promotion - cast explicitly with `as` if you need to mix types.

    +  -  *  /              Int      -> Int
                            Double   -> Double

    ==  !=                  Int      -> Bool
                            Bool     -> Bool

    <  >                    Int      -> Bool
                            Double   -> Bool

Equality (== / !=) is not defined on Double on purpose: comparing floats
for exact equality is almost always a bug. Use `<` / `>` to check
ordering, or wrap the comparison yourself when you really mean it.


Casts
-----

The `as` keyword converts between primitive types. Only these
directions are permitted:

    Int    as Double        widens to f64
    Double as Int           truncates toward zero
    Int    as Bool          0 becomes false, anything else true
    Bool   as Int           false becomes 0, true becomes 1

Bool and Double cannot be cast in either direction. Going from Bool to
Double has no useful purpose (write `1.0` / `0.0` directly), and Double
to Bool has no clean semantics (-0.0 and NaN have non-zero bit
patterns), so the typechecker rejects both. Cast through Int if you
really need the conversion.
